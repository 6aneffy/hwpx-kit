# format-convert: 표기 변환 (금액·날짜·만나이)

**이 변환들은 절대 손(암산)으로 하지 말 것** — 큰 금액의 한글 자릿수, 요일,
만나이는 미세하게 틀리기 쉽다. 반드시 CLI를 호출한다 (결정론, 안 틀림):

```
hwpx-kit fmt --amount 12340 --json                → 금12,340원(금일만이천삼백사십원)
hwpx-kit fmt --amount 12340 --style ilgeum --json → 일금 12,340원정(일금 일만이천삼백사십원정)
hwpx-kit fmt --date 20260101 --json               → 2026.1.1.(목)
hwpx-kit fmt --age 930705 --base 20260710 --json  → 930705(32)
```

JSON 봉투의 `data.result`를 그대로 사용. 종료코드 0 성공 / 1 오류(사유는 error.message).

**값이 여러 개면 한 셸 호출로 묶어라** (`;`로 연결하거나 루프) — 건별로
따로 실행하면 승인 프롬프트와 왕복만 늘어난다.

## 금액 스타일 선택 — 문서 맥락으로 판단

| 스타일 | 형식 | 쓰는 곳 |
|---|---|---|
| `gongmun` (기본) | 금12,340원(금일만이천삼백사십원) | 정부·지자체 공문, 지출결의, 공고문, 행정 계약 문서 — 법정 형식(행정 효율과 협업 촉진에 관한 규정 시행규칙) |
| `ilgeum` | 일금 12,340원정(일금 일만이천삼백사십원정) | 영수증, 차용증, 민간 계약서 — '정(整)'으로 뒤에 숫자 못 붙이게 하는 민간 관습 |

작성 중인 문서가 공문서면 gongmun, 민간 서류면 ilgeum. **애매하면 사용자에게
어느 형식인지 확인**하고, 문서에 이미 쓰인 금액 표기가 있으면 그 형식을 따른다.

## 규칙 요약 (검증·설명용 — 계산은 여전히 CLI로)

- 금액 한글 읽기는 위변조 방지 표기: 십/백/천 앞 '일'을 생략하지 않음
  (113,560 → 일십일만삼천오백육십 — "십일만"이 아님)
- 날짜: 월·일 앞 0 없음, 일 뒤에도 마침표, 요일은 괄호 — `2026.1.1.(목)`
- 만나이: 주민번호 앞 6자리는 19YY로 해석. `--base` 생략 시 오늘 기준

## 문서에 넣기

변환 결과는 hwpx-form 스킬의 fill 값으로 사용한다:
`{"table:계약금액": "<fmt 결과>"}` 또는 `{"text:<원문 금액>": "<fmt 결과>"}`.
