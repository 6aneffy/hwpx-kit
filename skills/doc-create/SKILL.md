# doc-create: 백지에서 한글 문서 만들기

문서 생성 요청이면 **다른 무엇보다 먼저 — 파일 탐색·폴더 뒤지기 전에 —
AskUserQuestion으로 확인** (탐색은 토큰 낭비, 답이 "있음"일 때만 찾는다):

> "참고할 양식(hwpx/hwp) 파일이 있나요?"
> 1. 있음 — 파일 경로를 알려주거나 대화에 끌어다 놓기
> 2. 없음 — 백지에서 새로 만들기

이 질문에 hwpx-form의 정보 게이트 질문(제목·주최기관·**장소**·일시·대상·정원·
구성)을 같이 묶어서 물으면 왕복이 준다 — **장소를 빼먹지 말 것** (실사용
테스트에서 누락된 항목).

## 양식이 있다면 → hwpx-form 흐름

경로를 받아 hwpx-form 스킬의 흐름(신작이면 정보 게이트부터)으로 진행.
.hwp면 먼저 `hwpx-kit convert`.

## 양식이 없다면 → 생성 경로 (품질 순)

**번들 템플릿은 계획서·기획서 계열용이다** — 만들 문서가 그 계열이면 1번,
다른 유형(신청서·보도자료·공문 등)이면 템플릿을 억지로 쓰지 말고:
해당 기관에서 쓰는 양식 파일을 구해다 달라고 먼저 권하고, 없다면 2번
(generate)으로. 계열 판단이 애매하면 사용자에게 물어본다.

1. **번들 템플릿** (계획서·기획서 계열 최우선): 이 스킬 파일의 베이스 디렉토리에서 두 단계 위
   `templates/Default_Template.hwpx` (플러그인 루트/templates/) — 범용
   계획서 틀: 표지(0000·주최·주관 자리표시) + 장 헤더 박스(번호 빈칸) +
   추진배경·개요 골격 + 빈 커리큘럼형 표(21행) + 예산표 골격.
   **본문 골격(○·□·- 문단)에는 `{{항목N}}`/`{{소제목N}}`/`{{세부N}}` 마커가
   심어져 있다** — analyze의 marker: 키로 정밀 채우기 (같은 ○끼리 text:로는
   구분 불가하기 때문). 채우는 순서:
   - **채우기 전에 초안 확정** — 커리큘럼 구성·예산 항목/수치처럼 사용자가
     주지 않은 내용은 요약해 보여주고 승인받은 뒤 진행 (임의 수치로 바로
     채우지 말 것)
   - **장 복제 먼저** (마커가 앵커로 쓰이니 채우기 전에):
     `table-copy`로 장 헤더 표 복제 — 개수 제한 없음, 앵커는 직전 장의 마커 문단
   - 장 번호·제목: table-set (번호 셀 (0,0), 제목 셀 (0,2))
   - 표지: 제목("0000")·부제("계획(안)")·연월("2026. 00.")은 text: 교체,
     주최·주관은 `marker:주최주관`. **자리표시는 사용자가 준 텍스트로 통째
     교체** — "주최·주관: ○○" 처럼 라벨을 덧붙이지 말 것 (자리표시 = 그
     자리에 들어갈 최종 문구의 견본일 뿐) / 개요: text: 교체 / 본문: marker:
   - **장 시작 쪽나눔은 템플릿에 내장돼 있음** (전 장) — 기존 장은 신경 쓸
     필요 없고, **table-copy로 끼워넣은 장만** `page-break --table`로 처리
   - **표지는 조정 불필요** — 템플릿에 쪽나눔이 내장되어 제목이 1~3줄이어도
     본문(1장)이 항상 새 쪽에서 시작한다. 제목이 4줄 이상으로 극단적으로
     길면 사용자에게 한글에서 표지만 손보라고 안내
   - **안 쓰는 마커 문단은 delete:로 제거** — 원문은 read 출력에서 그대로
     복사 (마커 앞 공백이 미묘해서 손으로 치면 빗나간다)
   - **예산표 채우기 요령**: 비목·항목명은 셀 한 줄에 들어가게 짧게(6자 내외
     — 길면 행이 부풀어 총계가 다음 쪽으로 밀린다). 잔여 빈 행은 두고
     "필요 없으면 한글에서 행 삭제" 안내
   - 안 쓰는 골격 표(커리큘럼·예산)는 table-clear로 비우거나 한글 삭제 안내
   - 다른 크기 표가 필요하면 `table-new --rows R --cols C --like-table <기존표>`
     (기존 표 서식 차용). 표 상태 검사는 `table-map` (원시 XML 파싱 금지)
   서식 품질이 보장되는 유일한 경로. 템플릿 파일이 없으면 2번으로
2. **generate 명령**: 내용을 Markdown으로 작성 후
   `hwpx-kit generate <초안.md> --out <문서.hwpx> --preset 보고서 --json`
   → validate → 한글에서 확인 안내. 기본 공문서 스타일이라 표지 디자인·
   색 헤더박스 같은 꾸밈은 안 나온다 — 이 한계를 사용자에게 미리 알리고,
   "서식이 중요하면 기관에서 쓰는 양식 파일을 구해다 주시는 게 최선"이라 안내.
   (generate는 kordoc 필요 — 없으면 오류 메시지의 설치 안내를 사용자에게 전달)

어느 경로든 내용 작성 전에 hwpx-form의 정보 게이트(제목·기관·일시·장소·
대상·구성)를 그대로 적용 — 최소 충족 후 시작. 글머리 위계·표기는
gongmun-format 스킬, 금액·날짜는 format-convert 스킬을 따른다.

## 주의

- 사용자에게 특정 폴더 구조(incoming/ 등)를 가정하지 말 것 — 파일 위치는
  항상 물어서 받는다
- 백지 생성물도 원본 개념이 없을 뿐 결과물 관리는 동일: 사용자가 지정한
  위치에 저장하고 경로를 알려준다
