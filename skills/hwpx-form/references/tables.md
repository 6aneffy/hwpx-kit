# 표 만들기·조작

## 표 그려줘 — "이렇게 표 만들어줘", "이 이미지 표처럼"

새 표는 원시 도구를 이어 붙이지 말고 **table-build 스펙 하나로 원샷**:

1. **구조를 먼저 스펙으로 옮긴다** (만들면서 정하지 말 것): 행×열, 병합
   목록(시간표의 연강 칸·세로 구분 칸), 열폭 비율(시간 열은 좁게), 헤더 행
2. **내용을 칸에 맞게 압축** — 참조 이미지의 긴 설명은 핵심만. 칸 폭 대비
   긴 문장을 그대로 부으면 행이 부풀고 선에 붙는다 (실증된 품질 저하 1순위)
3. 스펙 JSON 작성 후:
   `hwpx-kit table-build <문서> --spec <스펙.json> --at-text "<앵커>" --out <결과>`
   ```json
   {"rows": 12, "cols": 6, "col_widths": [1,2,2,2,2,2],
    "header_rows": 1, "header_color": "#D9E5FF", "align": "center",
    "font_pt": 10,
    "merges": ["1,1:2,1", "3,0:4,0"],
    "cells": {"0,0": "시간", "0,1": "3. 23.(월)", "1,1": "이동"}}
   ```
   — 병합·열폭·헤더 색/볼드·내용·**가운데 정렬**까지 명령 한 번
   - 표는 자동으로 **가용 페이지 폭 전체**를 쓴다 (좁은 기본 폭 잘림 해소)
   - `font_pt`(선택): 셀 글자 크기. 생략해도 문서 기본 글자가 과대(16pt 초과,
     제목 위주 소스 등)면 자동으로 10pt로 줄여 잘림을 막는다. 촘촘한 표는
     `font_pt: 9`, 여유 있으면 생략(문서 기본 유지)
4. `table-map`으로 병합·내용 검증 → validate → 한글 육안 안내
5. 못 미친 부분만 원시 도구(cell-merge/color/align/col-width)로 후속 수정

기본값 관습: 시간표·현황표는 `align: center` + 헤더 색(원본 있으면 그 색,
없으면 옅은 파랑 #D9E5FF 계열). 본문형 셀(설명 문단)은 정렬 생략(왼쪽).

복붙(서식 깨짐) 대신 도구 조합으로:
1. `hwpx-kit read <파일.xlsx> --format md` — 표 데이터 확보
2. `outline`으로 삽입 위치(앵커 문단/표) 확인, 애매하면 사용자에게
3. `table-new --rows R --cols C --like-table <이웃 표> --after-text "<앵커>"` —
   문서 기존 표 서식을 차용해 새 표 생성 (이웃 표 없으면 --like-table 생략)
4. `table-set --data <셀.json>`으로 값 기입 — 숫자는 세 자리 콤마 등 표기는
   table-calc 스킬 관습, 헤더 행은 (0,c) 좌표
5. validate → 경로 안내
