# table-calc: 표 수치 계산·표기

보고서 표의 수치 작업 = **읽기(read) → 계산 → 표기 규약 적용 → fill 착지** 흐름.
계산은 반드시 코드(파이썬 등)로 하고 암산하지 않는다. 큰 금액 한글 병기가
필요하면 format-convert 스킬(`hwpx-kit fmt`)과 조합.

## 증감 표기 관습 (행정 보고서)

전년/전월 대비 증감 열을 채울 때, 문서에 이미 쓰인 기호를 따르고 없으면 사용자에게 확인:

| 표기 | 규칙 | 예 (전년 100 → 올해 80 / 120) |
|---|---|---|
| `△` | **음수만** △ 붙임, 양수는 그대로 | `△20` / `20` |
| `▲▽` | 양수 ▲, 음수 ▽ | `▽20` / `▲20` |
| `↑↓` | 양수 ↑, 음수 ↓ | `20↓` 또는 `↓20` (문서 관례 따름) |
| `%` | 증감률 = (B−A)/A×100 | `△20.0%` / `20.0%` |
| `%p` | **비율끼리의 차이**는 %가 아니라 %p (예: 점유율 35%→40% = 5%p 상승) | `5%p` |

- 소수 자릿수는 표의 기존 값과 맞춘다 (기존이 소수 1자리면 증감도 1자리)
- 행정 관습에서 `△`는 감소(음수) 표시다 — 수학의 델타(변화량)로 오독하지 말 것

## 기타 표 유틸 (요청 시)

- **비율**: 각 행 값 ÷ 합계 × 100. 자릿수 맞춰 반올림하고, 반올림 후 합이 100.0이
  안 되면 최대 항목에서 조정 후 "반올림 조정" 사실을 사용자에게 알림
- **순번**: 좌상단부터 1, 2, 3… (`table:` 키 + `#N`으로 각 셀 지정)
- **콤마**: 세 자리마다 `10000 → 10,000` — 표 안 숫자 전체 일괄이면 read로 원문
  수집 후 `text:` 키 다중 교체
- **역합계**: 합계 칸이 이미 있고 항목이 빈 경우 — 합계에서 역산해 배분하되
  근거를 사용자에게 확인 (임의 배분 금지)
- **검산**: 채우기 전에 합계 행이 항목 합과 일치하는지 확인, 불일치하면 채우지
  말고 사용자에게 보고

## 착지 (문서에 쓰기)

계산 결과는 hwpx-form 스킬의 fill 계약으로 셀에 넣는다 — 새 방식을 만들지 말 것:
- 라벨 오른쪽 칸: `table:라벨`, 중복 라벨: `table:라벨#N`, 다른 방향: `> down`/`> right` 체인
- 이미 값 있는 셀 교체: 그 셀 원문으로 `text:<원문>` 키
- 원본 불변 — `--out` 사본, 완료 후 `validate`
